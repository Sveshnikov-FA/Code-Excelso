The R tutorial

R – это язык программирования, разработанный в 1993 для статистического и графического программирования. Несмотря на то, что его название состоит из одной буквы, R способен на очень многое, от простых математических вычислений к комплексным программам по машинному обучению и биоинформатике. Его синтаксис столь эффективен и прост, что любые его недостатки уходят на второй план.
Перед тем, как начать писать код, мы должны убедиться, что у нас установлено всё необходимое. Нам будут нужны компилятор, который можно скачать на официальном сайте языка, и среда разработки, роль которой может выполнять RStudio.

Для программистов уже стало традицией начинать программировать на новом языке с вывода в консоль сообщения “Hello, World!”. Приступим же к делу.

Код
Результат
print ("Hello, World!")
cat (“Also hello, but with variables”,var_x)

Даже эти столь простые строки кода могут многое нам сказать о языке: данные печатаются с помощью функции print, а сами данные в скобках и кавычках. Точки с запятой в конце строк не ставятся. Cat – почти то же самое, что и print, но эта функция собирает несколько типов данных и переменных в один вывод, в то время как print печатает только одну строку.
На переменные в R стоит обратить особенное внимание. Типы обыкновенные – целое и действительное число, строка, и булева переменная. Есть три возможных способа объявить переменную и присвоить ей значение: с помощью операторов равно, лево и право.

var.1 = 5
var.2 <- "R"
TRUE -> var.3

Чтобы считать пользовательский ввод в переменную, мы используем функцию readline() с параметром prompt, означающим вывод строки рядом с местом ввода вот так:

my.name <- readline(prompt="" )

Объявление массивов в R предельно просто. Единственная сложность – это поиск отличий между массивоподобными структурами данных. Vector – обычный массив, для объявления которого нужно лишь уточнить его название, использовать оператор присвоения и перечислить элементы, объединённые функцией с(). Однако размер такого массива не меняется до конца выполнения программы. В случае с list его содержимое может отличаться по типу между собой, его размер можно изменить, и частью его содержимого может быть вектор. Для объединения элементов используем функцию list ().

Vector_x <- c( 'red', 'green', "yellow")
list1 <- list(c(2,5,3),21.3)

Условия в R работают так же просто. Если некоторое условие выполняется, выполняется соответствующее действие.

if (condition) {
do a thing
} else if (condition && condition || condition) {
do a thing
} else {
do a thing
}

Циклы – отличная вещь. Они делают нашу жизнь проще, прямо как умножение в математике. Есть 3 типа циклов: for, с которым можно уточнить, сколько именно раз будет выполняться цикл; while, который продолжает выполняться пока выполняется некоторое условие; и repeat, бесконечный цикл.

for(i in 1:10) {}
while (response!=42) {}
repeat {}

Зачем нам бесконечный цикл, спросите вы? Ну, они имеют смысл, когда мы используем утверждения контроля циклов next, который пропускает оставшийся в цикле код и переходит на следующий круг, и break, который полностью прекращает дальнейшее выполнение цикла.
Теперь перейдём к созданию проекта. Учитывая, что R часто используют для машинного обучения, давайте же сделаем hello world машинного обучения: анализ датасета ирисов. Датасеты – буквально наборы данных, собранных в таблицу с соответствующими значениями. Он состоит из 50 разных результатов анализов 3 видов цветка ириса, со столбцами со значениями о длине и ширине пестика и лепестков и соответствующими названиями видов. Этот проект может показаться несколько страшным для начала, но на самом деле это далеко не так. Поехали! Для того, чтобы загрузить датасет, достаточно объявить его в переменной и переименовать для удобства:

#подключаем необходимую библиотеку install.packages("caret")
# подключение датасета к среде
data(iris)
# переименовываем датасет
dataset <- iris

Для получения его размеров, используем функцию dim:

Код
Результат
dim(dataset)

Для получения первых 5 рядов таблицы, используем функцию head:

Код
Результат
head(dataset)

Для получения процентного соотношения каждой группы датасета, напишем следующий код:

Код
Результат
percentage <- prop.table(table(dataset$Species)) * 100
cbind(freq=table(dataset$Species), percentage=percentage)

Функция cbind, использованная выше, отвечает за создание таблицы с двумя столбцами: freq, то есть вид ириса и процентное соотношение данных из каждого последующего столбца.
А для получения общего отчёта о датасете,как например, среднее, максимальное и минимальные значения ряда, используем функцию summary:

Код
Результат
summary(dataset)

Теперь к обучению. Мы будем использовать 10-ступенчатую кроссвалидацию для получения максимальной точности. Сложные слова, я знаю. Проще говоря, мы разделим наш датасет на 10 частей: анализируем и обучим на девяти, а в одну выведем полученное предсказание данных. Мы повторим процесс 3 раза для каждого алгоритма, с различным рвзделением и тестированием данных на 10 частей, чтобы получить более точный результат. TrainСontrol – функция, которая обучает набор параметров, используя выбранный алгоритм.

control <- trainControl(method="cv" , number=10)

metric <- "Accuracy"

Теперь мы вычисляем, какой алгоритм работает лучше всего. Выбираем из: Linear Discriminant Analysis (LDA) , Classification and Regression Trees (CART) , k-Nearest Neighbors (kNN), Support Vector Machines (SVM) или Random Forest (RF). Самый очевидный способ - протестировать их все:

# a) linear algorithms
set.seed(7)
fit.lda <- train(Species~., data=dataset, method="lda" , metric=metric, trControl=control)

# CART
set.seed(7)
fit.cart <- train(Species~., data=dataset, method="rpart" , metric=metric, trControl=control)

# kNN
set.seed(7)
fit.knn <- train(Species~., data=dataset, method="knn" , metric=metric, trControl=control)

# SVM
set.seed(7)
fit.svm <- train(Species~., data=dataset, method="svmRadial" , metric=metric, trControl=control)

# Random Forest
set.seed(7)
fit.rf <- train(Species~., data=dataset, method="rf" , metric=metric, trControl=control)

Заметная строка set.seed(7) используется для обновления процесса, что в свою очередь, обеспечивает генерацию более похожих между собой чисел, используемых для тестирования алгоритма. Сверим полученные результаты с помощью функции summary:

Код
Результат
results <- resamples(list(lda=fit.lda, cart=fit.cart, knn=fit.knn, svm=fit.svm, rf=fit.rf))
summary(results)
