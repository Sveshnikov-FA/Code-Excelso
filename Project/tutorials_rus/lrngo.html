    Go, или GoLang – язык программирования, разработанная командой из Google, когда они захотели упростить сообщения между сетями и сделать разработку приложений более простой, а сами приложения – производительнее. Как следует из названия, команда разработчиков продолжает двигаться вперёд и постоянно ищет способы сделать язык лучше. Здесь мы будем работать с версией 1.10 языка. В основном язык используется для вычислений, серверного и десктоп- программирования. У него есть свои минусы, как местами путанный синтаксис, и свои плюсы, как лёгкость в изучении. Если можно сказать что-то точно, то учитывая тот факт, что этот язык поддерживается никем иным, как Google, можно точно сказать, что этот язык в ближайшее время не потеряет свою актуальность.
Перед тем, как мы начнём программировать на Go, мы должны убедиться, что у нас установлено всё необходимое, как компилятор, доступный для скачивания на официальном сайте, и среда разработки, наиболее подходящей из них может быть GoLand, доступный для скачивания на сайте JetBrains.

Для программистов уже стало традицией начинать изучать новый язык с вывода в консоль “Hello World”. Приступим же!

Код
Результат
import “fmt”
func main() {
fmt.Println(“Hello World”)
}

Разберём же этот код. С помощью Import мы импортируем внешние библиотеки и модули. Fmt – библиотека, отвечающая за вывод в консоль в Go. Func – кодовое слово для объявления функций в Go, где main – её название, а аргументы, значения пересылаемые в функцию, помещены в скобки (в данном случае их нет). С функции main начинается выполнение кода. Println – метод, отвечающий за вывод в консоль. Также стоит заметить, что в конце строк в go точка с запятой не ставится.
Для принятия пользовательского ввода мы должны импортировать bufio и os, а затем объявить считыватель. Должно получиться следующее:

import (
"bufio"
"fmt"
"os"
)
func main() {
reader := bufio.NewReader(os.Stdin)
text, _ := reader.ReadString('\n')
}

В Go переменные объявляются с помощью var, почти без объявления типов. Также можно вернуться к временам паскаля и объявлять переменные с помощью оператора := .

Var a = “init”
Var b = 1
var h,I = 5, 25
var g int
C := “init”
d := 5

Также стоит заметить, что Go крайне прогрессивен со своими переменными: если уточнён тип переменной, но не значение, то компилятор автоматически ставит им значение 0 или «». То есть в коде выше, g = 0.
Условия в Go достаточно прямолинейны.

if condition {
thing
} else if (condition2 && condition3) {
thing
} else {
thing
}

Прямо как в любом другом языке, с которым вы можете быть знакомы. Единственная разница – скобки для условий необязательны.
Циклы – полезная вещь. Они выполняют код снова и снова за нас. Go вновь продвигает инновации: здесь есть только 1 вид циклов: for. Но у этого вида много разных вариантов применений:

I:=1
for I <=3 {
thing
I = I + 1
}
For j:=7; j <=9; j ++ {
thing to repeat
}
for {
thing
break
}

Бесконечные циклы просто ломают код. Точнее, ломали бы, если бы не было утверждений контроля циклов. Есть два вида: break, которое полностью прекращает выполнение цикла, continue, которое просто пропускает оставшийся код цикла и сразу идёт на следующий круг.
В Go, массив – пронумерованный набор элементов соответствующей длины. Объявляются они так:

var a [5]int
b := [5]int{1, 2, 3, 4, 5}

Где 5 - длина массива, int – тип элементов, а в фигурных скобках – сами элементы. Для получения соответствующего элемента нужно поместить его порядковый номер в квадратные скобки рядом с названием массива, а для получения длины массива используем метод len.

Len(arr)
arr[2]

Теперь соберём полученные знания и создадим проект. Пусть это будет простая игра в понг, и она будет размещена на сервере. Для этого нам понадобится компилятор GopherJS и библиотека GoDOM, которая была создана для упрощения работы с перым и добавления некоторых функций, которые имеют больше общего с JS. Заметьте, что я сказал, что GopherJS – компилятор. Это означает, что мы просто пишем немного кода на Go, а затем во время сборки кода компилятор конвертирует написанное в JS-файл. Но начнём же делать сам проект: создадим базовую страницу, на которой и будет размещена наша игра. На ней должно быть два блока, мяч, разделитель и очки. Не забудьте оставить ссылку на автоматически сгенерированный файл JS, чтобы наша игра вообще работала. Так должен выглядеть наш HTML-код:

< div id="block"> < /div>
< div id="block_r"> < /div>
< div id="ball"> < /div>
< div id="container_1">
< div id="score">0 < /div>
< div id="score_r">0 < /div>
< /div>
< div id="divider"> < /div>
< script src="script.js"> < /script>

Я таже добавил отдельный скрипт, который удерживает правый блок на месте:

< script>
var b = document.getElementById("block_r");
setInterval(function () {
b.style.left = window.innerWidth - 35 + "px";
},100)
< /script>

Теперь перейдём к написанию кода на Go. Для начала объявим все необходимые переменные:

< link rel="stylesheet" href="main.css">

Moving onto the script-writing phase. First of all, we should declare some variables that are going to be frequently used in func main(), like the dimensions of the screen and ball coordinates.

var iw, ih= js.Global.Get("innerWidth").Int(), js.Global.Get("innerHeight").Int()
var left, top= iw/2, ih/2
ball := Document.QuerySelector("#ball")
f := Document.QuerySelector("#block")
g := Document.QuerySelector("#block_r")
scA := Document.QuerySelector("#score")
scB := Document.QuerySelector("#score_r")
scoreA := scA.Get("innerHTML").Int()
scoreB := scB.Get("innerHTML").Int()
dir := 1
dirY := 1
var iter= false

Разберём же некоторые части этого кода. js.Global - то же самое, что и document.body в JavaScript. А что тогда значит Document? Это тоже document.body, просто первое – это синтаксис GopherJS, а последнее – GoDOM. Я знаю, несколько запутанно, но так будет лучше, поверьте. QuerySelector для GJS – то же самое, что и document.getElementBbyId(), лишь с той разницей, что QS работает и на тэгах, и на классах, и на id. Переменные scoreA и B были созданы для того, чтобы следить за очками в игре. Переменные dir отвечают за направление движения мяча.
Теперь создадим функцию, работающую периодически, и которая не помешает правильнову выполнению кода, идущему после неё. Для этого нам понадобятся goroutines, которые имеют много общего с обычными функциями, но эти выполняются одновременно с кодом, идущим после неё, как бы на втором плане. Затем мы помещаем в неё бесконечный цикл, и если бы не goroutines, код после этого не был бы выполнен никогда. Первым делом обновим наши переменные, а именно размеры экрана, координаты мяча и блоков.

go func() {
for true {
iw = js.Global.Get("innerWidth").Int()
ih = js.Global.Get("innerHeight").Int()
topA := f.Get("style").Get("top").Int()
topB := g.Get("style").Get("top").Int()
if iter {
left = ball.Get("style").Get("left").Int()
top = ball.Get("style").Get("top").Int()
}

Теперь поработаем с логикой игры. Если мяч ударился о границу окна или блок, то он должен поменять направление движения, и очки должны начислиться соответственно.

if left > iw-23 || left < 0 || (left <=3 2 && top>= topA-5 && top <=t opA+125) || (left>= iw-45 && top >= topB-10 && top <=t opB+125) {
if left <=0 {
scoreB = scoreB + 1
scB.SetInnerHTML(strconv.Itoa(scoreB))
} else if left >= iw-23 {
scoreA = scoreA + 1
scA.SetInnerHTML(strconv.Itoa(scoreA))
}
dir = dir * -1
}

Не стоит забывать и о верхних границах!

if (top > ih-23 || top < 0) {
dirY = dirY * -1
}

Возможно вы заметили, что чуть ранее я добавил .Int() после использованияinnerHTML, а теперь ещё и какой-то strconv появился. Дело в том, что innerHTML приходит в скрипт в виде строки, а так как мне нужно складывать и вычитать эти значения, я должен конвертировать эти значения в число. При пересылке обратно мы должны сделать обратное, с чем нам и помогает библиотека strconv.
Теперь проработаем движение мяча. Вот здесь-то нам и пригодились переменные dir! Для перемещения мяча мы просто прибавляем их к его нынешним соответствующим координатам, а переменные могут быть равны только либо 1, либо -1.

ball.Get("style").Set("left", strconv.Itoa(left+dir))
ball.Get("style").Set("top", strconv.Itoa(top+dirY))

Затем мы меняем значение iter для дальнейшего обновления координат мяча, и ставим небольшую задержку для лучшего движения с помощью метода sleep.

iter = true
time.Sleep(time.Nanosecond)
}
}()

Наконец, мы должны сделать так, чтобы блоки двигались, когда мы нажимаем клавиши “W” “S”, “вверх ”и “вниз” с помощью функции, выполняемой каждый раз, когда нажата клавиша на клавиатуре. Для проверки нам понадобятся коды клавиш - 87, 83, 38 и 40 соответственно. Также нужно проверить, не слишком ли низко или высоко расположен блок: если да, то двигаться он не должен.

Document.AddEventListener("keydown", func(e Event) {
var tp= f.Get("style").Get("top").Int()
var tp1= g.Get("style").Get("top").Int()
if e.KeyCode() == 38 && tp > -5 {
f.Get("style").Set("top", (strconv.Itoa(tp-10) + "px"))
}
if e.KeyCode() == 40 && tp < (ih-125) {
f.Get("style").Set("top", (strconv.Itoa(tp+10) + "px"))
}
if e.KeyCode() == 87 && tp1 > -5 {
g.Get("style").Set("top", (strconv.Itoa(tp1-10) + "px"))
}
if e.KeyCode() == 83 && tp1 < (ih-125) {
g.Get("style").Set("top", (strconv.Itoa(tp1+10) + "px"))

})
