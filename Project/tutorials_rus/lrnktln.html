Kotlin – достаточно новый язык программирования, разработанный JetBrains, для того чтобы быть безопаснее и эффективнее, чем Java и проще, чем Scala. В основном используется для разработки десктоп и мобильных приложений. Возможно, скорость выполнения программ может быть местами разочаровывающей, Kotlin блестяще выполняет поставленные задачи.
Для начала нам стоит убедиться, что у нас установлено всё необходимое, как, например, компиятор, доступный для скачивания на официальном сайте, и среда разработки, роль которой может выполнить IntelliJ IDEA.

Для программистов уже стало традицией начинать программировать на новом языке с вывода в консоль сообщения “Hello, world!”. Приступим же!

Код
Результат
fun main(args: Array < String>) {
println("Hello World!")
}

Разберём этот код. Fun - кодовое слово для объявления функций в Kotlin, main – её название. В скобки мы помещаем аргументы, то есть переменные со значениями, пересланными в функцию из места вызова. В случае с функцией main, аргументы должны выглядеть только так, и никак иначе. Действия, выполняемые функцией помещаются в фигурные скобки, но точка с запятой в конце строк не ставится.
Теперь перейдём к переменным. Более современные языки уходят от правила об обязательном объявлении типа, и Kotlin – не исключение. Ну, то есть при желании конечно же можно уточнить тип переменной, но это совсем не обязательно.

val firstName: String = "fName"
и
val firstName = "fName"

делают абсолютно одно и то же. С помощью val мы объявляем константы, а для объявления переменных нужно использовать var. Но что нам делть, если мы хотим присвоить очень большое число как значение новой переменной? Что же, необязательно возвращаться к полному объявлению типа переменной, ведь можно использовать сокращённую форму:

val myLong = 19L

Напоминаю, что этот код работает так же, как и код с полным объявлением типа:

val myLongAgain: Long = 40

Ещё одно интересное свойство переменных в Kotlin – это то, что в числа можно ставить нижние подчёркивания, игнорируемые при вычислении. Возможно, это сделает большие числа более удобными для чтения.

val oneMillion = 1_000_000

Массивы – пронумерованная последовательность элементов некоторой длины. Для объявления такого мы должны использовать функцию arrayOf и перечислить элементы в скобках, вот так:

val myArray = arrayOf(1, 2, 3, 4, 5)

Выглядит достаточно просто, не правда ли? Тогда вот что-то менее понятное:

val myArray = arrayOf(1, 2, 3, 4, 5, “what”)

Этот код скомпилируется без ошибок. Такой массив называется массив смешанного типа. Для точности можно уточнить тип элементов в угловых скобках.

val myArray = arrayOf < Int>(1, 2, 3, 4, 5)
//или
val myArr = intArrayOf(1,2,3,4,5)

Теперь посмотрим, как принять пользовательский ввод. Рассмотрим два самых практичных способа. Первый – это использовать столь знакомую нам функцию readline(), которая работает… вот так?

val (a, b) = readLine()!!.split(' ')
println(a.toInt() + b.toInt())

Это означает, что readline принимает целую строку, которую для распределения между двумя переменными нужно арзделить на две на месте пробела. Что ж, к такому синтаксису нужно будет привыкнуть. Вот ещё один способ принятия ввода.

import java.util.Scanner
fun main(vararg args: String) {
val input = Scanner(System.`in`)
val a = input.nextInt()
val b = input.nextInt()
println(a + b)
}

Этот выглядит гораздо опрятнее, но он требует импорта модуля, что может запутать при работе с более глобальным проектом.
Условия в Kotlin работают достаточно простым образом:

if (condition) {
do a thing
} else {
Do a slightly different thing
}

Циклы – отличная вещь. Они помогают нам избавиться от лишней монотонной работы. Есть два типа циклов: for и while. С for мы точно указываем диапазон, в котором работает переменная цикла, или итератор. Вот несколько способов объявления цикла for: Простой диапазон:

for (i in 1..3) { println(i) }

Диапазон от большего к меньшему с изменённым шагом:

for (i in 6 downTo 0 step 2) { println(i) }

Диапазон в массиве, идущий от 0 до индекса последнего элемента.

for (item in collection) { print(item) }

А цикл while просто продолжает выполняться, пока выполняется некоторое условие. Никаких итераторов, никаких шагов.

while (condition) { do a thing }

Иногда циклы слишком прекрасно работают, и они перестают нам быть нужны. Для такиъх случаев есть утверждения контроля цикла, как break, которое полностью прекращает выполнение цикла, и continue, которое лишь пропускает оставшийся код в цикле и сразу переходит на следующий круг (или итерацию).
Давайте же попробуем сделать проект! Мы разработаем десктоп-приложение о кофе, мы же программисты, как-никак. Если вы просматривали мой курс по Java, то этот проект будет звучать и выглядеть знакомо, как и всё, связанное с Kotlin. Для начала настроим наше окно. Для этого нам понадобятся scene и stage, которые отвечают за рамку вокруг окна и содержимое соответственно. Настройка окна будет проходить в отдельном классе. Затем мы запускаем окно, используя соответствующую команду и заканчиваем настройку, изменив свойства окна, такие как его размер, видимость, фон, возможность изменять размер. Должно в итоге получиться что-то похожее на код ниже:

Код
Результат
class Main: Application() {
override fun start(stage: Stage) {
var scene: Scene = Scene(root, 955.0,700.0, Color.WHITE)
scene.root = root
stage.scene = scene
stage.title = "coffee game v2"
stage.isResizable = false
stage.show()
} }
fun main(args: Array ) {
Application.launch(Main::class.java, *args)
}

Можно сказать, что функция start - это наша сцена действий. Теперь объявим необходимые переменные, например массив ингредиентов, служебная переменная, название вида кофе и картинки. Конечно, у меня очень много ингдедиентов и возможных результатов, поэтому я сильно укорочу свой код, для удобства в чтении и для того, чтобы читатели не заснули, читая одну и ту же строчку десятый раз подряд.

var coffee: MutableList = mutableListOf()
var wh = 0
var lab = Label("")
lab.font = Font.font("Monospaced",32.0)
lab.translateY = 50.0
var table = Image(Main::class.java.getResourceAsStream("table.png"))

После введения переменных для картинок, мы инициализируем их и устанавливаем их координаты на экране.

var tableInit = ImageView(table)
tableInit.x = 90.0
tableInit.y = 250.0

Kotlin даёт нам возможность не создавать отдельную функцию в функции в функции для чтения ввода действий с мыши и предоставляет нам удобный метод, позволяющий обновлять наш массив с ингредиентами вот так:

var lungoInit = ImageView(lungo)
lungoInit.x = 550.0
lungoInit.y = 219.0
lungoInit.setOnMouseClicked {
coffee.add("lungo")
}

После того, как мы инициализировали остальные объекты, настроим функции для кнопок. Когда мы нажимаем на кнопку «назад», наш массив с ингредиентами должен очиститься, служебные переменные должны быть равны 0, и всё ненужное должно быть удалено с экрана.

var backInit = ImageView(back)
var enterInit = ImageView(enter)
var root = Group(tableInit,lungoInit,kettleInit,whiskInit,milkInit,ristrInit,cpotInit,
enterInit,backInit,wcInit,teaInit,iceInit,lemonInit,lab)
backInit.x = 10.0
backInit.y = 10.0
backInit.setOnMouseClicked {
coffee.clear()
if(root.children.last()!=lab) root.children.remove(root.children.last())
wh = 0
lab.text = ""
}
