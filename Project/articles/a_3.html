A New Beginning

The world of programming languages is vast. When you get exposed to their busy realm, it’s easy to get swept away and swirled around in this whirlpool to the point of disorientation. To avoid this state of utter confusion, make sure to organize your learning routine.
I started my journey with Java. At the time, the technology and the incredible possibilities open to me which I never really used were mind-blowing. I went on to experiment more with it and created many a desktop application. The point I’m making here has nothing to do with bad MS Paint art I did for my games written in Java, but with the fact that to start off you need a framework to use. Knowing basic Java syntax has really helped me learn new languages and if you want to experiment with something new, it really doesn't take as much as one would think to be able to just pick up a language and say: “Alright, this might take some getting used to, but I can pick this up just like that”. And while knowing how to code in Java is a useful skill, I’d recommend to start off with something more general, like C++, perhaps.
After you make a couple of projects on your own in your framework language, at some point comes the time to finally let go and take a look at other languages. But here’s the problem: which one do you choose? Surely, considering the fact that I’m blowing this whole process out of proportion so much, it must be important, right? Well, it actually is important, just not as dramatically as one would imagine. So, to make the process even more simple, I went on to classify languages by application. (note: the classification is based on personal or widespread preferences) To start off, you have the general-purpose, very well-known and probably good for making apps on desktop, like the classic Java, the world-famous Python or the numeric C++. These also happen to be very good second or third languages to pick up before you go full-on industrial. If you’re still very new to programming, then you should learn these first, go at your own pace and not fill your head with these peculiar names listed below: after all these languages stand pretty well on their own. Then there’s languages that are good for mobile development, like the Java-bettering Kotlin and Scala or Swift, which is largely avoided by doctors. There are more industrial and scientific languages, like the useful for writing systems software Rust, the brave and new Julia, which is good for data analysis and plot building, the machine-school of a language that is R, the go-to analysis and calculation language F#. Another group of languages is a web-based one: the oh-so-convenient Ruby, the loud and proud Dart, the fast and fresh Golang, or the sophisticated Elixir. There are also languages for Game Development, like the simple GML or the formal and jacket-wearing C# or the flashy ActionScript. There are even languages for writing music composition, like ChucK or Reaktor.
So, now that you’ve opened up the documentation, rolled up your sleeves, cracked your knuckles a few times, and dig in, the question will soon hit you: Now what? Well, while the more sophisticated languages are a challenge to learn already, the more simple ones go better with a set of tasks to set for yourself. Quite a few good ones come from the 99-problems GitHub repo and many other similar ones, where you can find exercises ranging from pretty simple to rather difficult.
In conclusion, programming can be hard. Starting from scratch is even harder. But if you have even a distant idea of what you want to achieve, it’s… actually still going to be hard, but you’ll have a better time doing it!
