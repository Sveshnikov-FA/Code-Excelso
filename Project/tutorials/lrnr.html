The R tutorial

R is a programming language developed in 1993 for the purpose of statistical computing and graphics. Despite the simple single-character name, R is capable of quite a wide variety of possibilities, from simple numerical operations to advanced Machine learning and Bio Science. It’s just so efficient and easy-to-use that its certain problems with syntax can be overlooked.

Before we get to writing code, we have to make sure we have everything installed. You’ll need a compiler, the necessary tool which tells the computer what to do with the code, that’s available on the official R project website. And you’ll also need an IDE, an environment to write, execute and debug code, the role of which can be served by RStudio, available for free download here.

It’s a customary tradition for programmers to start their experience with a language by printing out the message “Hello, World!” in the console. So let’s just jump right into it.

print ("Hello, World!")

cat (“Also hello, but with variables”,var_x)

Even this simple string of code can tell us a lot about the language: you print out data with print and the data in question being in parentheses, you don’t put semicolons at the end of the line in this language. Cat is just like print, but it combines multiple items into a continuous print output, unlike print, which only prints a single item.

Variables in R are questionable. There are the usual types, like integer, string and boolean. To declare one, you don’t have to specify the type, but there are three ways to assign values – with an equal, leftward and rightward operators.

var.1 = 5

var.2 <- "R"

TRUE -> var.3

To read user input into a variable, just use the trusty readline() function.

my.name <- readline(prompt="")

Array declaration is rather simple in R. There are quite a lot of array-type data structures. To declare a vector, the basic array, you state the name, use the preferred assignment operator and then combine values with a c() function. To declare a list, an array type which can contain different types of variables, including vectors, you do the same as with vectors, but use list () instead of c ().

< p id="code"> Vector_x <- c('red','green',"yellow")

list1 <- list(c(2,5,3),21.3)

Conditionals in R are easy to define. If a certain condition is true, an action is executed. Their syntax works just like this:

if (condition) {

do a thing

} else if (condition && condition || condition) {

do a thing

} else {

do a thing

}

Loops are great. They make our life easier, much like multiplication in maths. There are three types of loops: for, with which you can define the first and last value of the loop control variable, while, which goes on while certain statement is true and repeat, which is an infinite loop.

for(i in 1:10) {}

while (response!=42) {}

repeat {}

But why would you ever need an infinite loop? Well, they make sense, when you use if with loop control statements, which are next, which skips the rest of the loop and moves onto the next iteration (name of statement checks out), and break, which completely stops the execution of the loop.

Now onto the project.Considering the fact that R is most used for machine learning, let’s just do the hello world of ML: iris dataset analysis. It might sound scary for something to start off, but it's most definitrely not. Let's go!

To load the dataset, simply define it and rename it for your project:

# attach the iris dataset to the environment

data(iris)

# rename the dataset

dataset <- iris

To measure its dimensions, we can use the dim function:

dim(dataset)

To get the first 5 rows of the data in the set, use the head function:

head(dataset)

To get the percentage of each group in the dataset, use the code below:

percentage <- prop.table(table(dataset$Species)) * 100

cbind(freq=table(dataset$Species), percentage=percentage)

And to get an overall summary of the data in the set, such as the minimum and maximum value and the amount of each property in the set, use the function summary:

summary(dataset)

To build a basic plot comparing properties of each kind of elements in the dataset, use the following code:

featurePlot(x=x, y=y, plot="box")

Now onto the learning part. We will use 10-fold crossvalidation to estimate accuracy. This will split our dataset into 10 parts, train in 9 and test on 1 and release for all combinations of train-test splits. We will also repeat the process 3 times for each algorithm with different splits of the data into 10 groups, in an effort to get a more accurate estimate. Train control is a function which trains a grid of particular parameters on slightly different data using a preferred algorithm.

control <- trainControl(method="cv", number=10)

metric <- "Accuracy"

Now we try to evaluate which algorithm is best for our task: Linear Discriminant Analysis (LDA) , Classification and Regression Trees (CART) , k-Nearest Neighbors (kNN), Support Vector Machines (SVM) or Random Forest (RF). The easiest way to find out is to test all of them:

# a) linear algorithms

set.seed(7)

fit.lda <- train(Species~., data=dataset, method="lda", metric=metric, trControl=control)

# CART

set.seed(7)

fit.cart <- train(Species~., data=dataset, method="rpart", metric=metric, trControl=control)

# kNN

set.seed(7)

fit.knn <- train(Species~., data=dataset, method="knn", metric=metric, trControl=control)

# SVM

set.seed(7)

fit.svm <- train(Species~., data=dataset, method="svmRadial", metric=metric, trControl=control)

# Random Forest

set.seed(7)

fit.rf <- train(Species~., data=dataset, method="rf", metric=metric, trControl=control)

The very noticeable and likely confusing set.seed(7) function is used because the train function uses sampling which is mostly dependant on RNG, which doesn’t work for the high-precision process of machine learning, whereas if we set the seed, the results are going to be more similar or even identical. Now we check our results by using the abovementioned summary function:

# summarize accuracy of models

results <- resamples(list(lda=fit.lda, cart=fit.cart, knn=fit.knn, svm=fit.svm, rf=fit.rf))

summary(results)

Or, if we want to see the difference properly, we can plot the results:

dotplot(results)

If we look at the results we'll see that LDA is clearly the best one. Now we make predictions, the cornerstone of machine learning. Confusion Matrix is a table of observed and predicted classes and results based on the input data, it being validationSpecies here.

predictions <- predict(fit.lda, validation)

confusionMatrix(predictions, validation$Species)
