The Julia tutorial

Julia is a rather common name. You might know a Julia yourself, but did you know that there is a programming language called that? Yes, indeed, Julia is a scientific, fast, new and fresh language mainly for numeric computing, but it’s also really good for general-purpose programming as well! It has a smooth learning curve, a friendly-looking syntax and a performance comparable to C. It takes the best qualities from some languages and meshes them together with numerical accuracy and an extensive mathematical function library, thus creating a language that’s ideal for data science.

To start working with Julia on your PC, you need, as always, make sure you have the compiler, a program that translates the written code into machine code, installed. You can download Julia for your OS here: https://julialang.org/downloads/ and then follow the instructions provided here: https://julialang.org/downloads/platform.html. To install Julia onto Jupyter Notebook, go to the Julia prompt that will pop up after proper installation, and type in:

Pkg.add(“IJulia”)

You can use the same line of code to install other external Julia packages, except instead of IJulia you type in the name of the desired library.

It’s a customary tradition when learning a new language for the first program to be the output of a sentence “Hello, World!” In Julia it’s rather self-explanatory.

println("Hello, World!")

As you might notice, there is no semicolon in the end of the line, which is a rule we shall follow. User input is the main means of user-program communication. There are many a way to get user input with Julia, but it is usually recommended to use the input() function. With input it’s also possible to send a prompt to the user, which should be placed in brackets, as shown below.

name = input("What is your name? ");

Variables are not a problem when learning Julia, considering the fact that there aren’t that many of them. When defining a new variable, you don’t need to declare a type, meaning you only need to type in the name of it and its value. The language knows what the variable type is, though, and to find it out you can use the function typeof(something).

a = 2

typeof(a)

Output: Int64

Same goes for declaring arrays, where you put its elements in square brackets.

data = [1, 2, 3, 4, 5]

To go a little further, to define a dynamic array, the size of which you can modify after declaration, you should specify the type like:

data = Int64[];

and to add an element you use push!(arrname, data).

push!(data, 1)

Let’s talk conditionals now. The general purpose of conditionals in programming is to perform an action if a certain set of conditions is met. The syntax in Julia for them works as follows:

if condition

action

elseif (condition && condition || condition)

action

else

action

end

Yes, elseif is a separate operator in Julia, it works just like else if does, but the latter is to be used as the final part of the conditional. && is used as logical and, and || is used as logical or.

You probably noticed the odd way actions are placed. That’s because of something called Indentation, which is a very important part of the Julia syntax. Because the curly brackets aren’t used in this programming language, the borders of code blocks are specified by the indentation from the left, one tab key press being equal to one level of indentation. To handle errors, it is recommended to add end at the end of the code block. Its proper application can be seen in the code snippets on this page.

Loops are great. They make our life better, our code more readable and our coffee stronger. While that last part may or may not be true, loops help improve our quality of code by iterating the necessary code on certain conditions. There are two types of loops: while, which iterates the code while a condition is true, and for which iterates the code a specific amount of times.

while condition

action

end

for i = 1:5

action

end

Just like with the conditionals, when defining the bounds or conditions for loops, you do so by simply typing them in after the declaring word. Also to note: instead of defining a whole new variable, and its bounds you can simply put in the range of the int in the loop, the starting and finishing points divided by a semicolon. In Julia, a function is an object that maps argument values to return something back. The basic syntax for defining functions in Julia is as follows, indentation rules still apply:

function f(x, y)

x + y

end

While this doesn’t have an explicit return statement, upon execution, the function does return the final result. But you can put a return statement to specify what you want to return in the end.

function g(x, y)

return x * y

x + y

end

That’s it about Julia for now.
