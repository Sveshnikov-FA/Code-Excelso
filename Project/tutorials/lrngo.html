The Go! tutorial

Go, or GoLang, is a programming language developed by a team at Google when they needed to simplify network interactions and make software development more efficient. Apt for the name, the development team just keeps going further in trying to find ways to make the language better, and they constantly update at such a rate that they are already planning to release Go v2.0. This particular article works with version 1.10 of the language. It’s mainly used for system and numeric coding and server-side programming. While aesthetically and practically syntax-wise it might turn some people off, but you just have to try it for yourself to properly find out whether this language is for you or not. Considering the fact that this language is backed by none other than Google, you can be sure that it’s not going to fade away and become irrelevant any time soon. To be fair, there are many reasons to like this language: it’s very efficient performance-wise, it has an active community which keeps making external packages, and it’s easy to use.
Before we get to code in Go, we need to install all the necessary things, like the Go compiler, available on the official Go Website, and the IDE, the best-suiting being GoLand by JetBrains, available for free. It’s customary for programmers to start learning a programming language by printing out “Hello World” in the console, so let’s just do exactly that.

import “fmt”
func main() {
fmt.Println(“Hello World”)
}

Let’s break this code down. Import “importname” is the way to import external libraries to access functions or data structures which aren’t in Go by default. Fmt is the library which helps with console output in go. Func is the basic definition of a function in Go, main being its name and arguments, variables that to send from the initial function being put in parentheses, but since main is, as the name says, an obligatory main function, from which the execution of the code actually starts, the argument list is to be always left empty. Println is the method which does the printing of the lines or variables. As you might know, the ln in the end means that the content printed is going to appear on a new line. Another thing to note is that statements in Go are left without any semicolons at the end, unlike many languages do. To get user input you need to import bufio and os, and then define the reader, the thing that will get input. Then you read the type like so:

import (
"bufio"
"fmt"
"os"
)
func main() {
reader := bufio.NewReader(os.Stdin)
text, _ := reader.ReadString('\n')
}

In Go variables are explicitly declared with var, with little to no type specification or you can go back to the good old days of Pascal and declare variables without var using the := syntax, but you can only declare multiple variables inline using var

Var a = “init”
Var b = 1
var h,I = 5, 25
var g int
C := “init”
d := 5

Also to note, Go is really progressive in its own way: if you declare a variable and specify the type with var but don’t specify the value, Go automatically gives it a value of either 0 or “”, depending on the type. In the code above, g is equal to 0. Conditionals in Go are rather straightforward.

if condition {
thing
} else if (condition2 && condition3) {
thing
} else {
thing
}

Just like in any language you might be familiar with. Well, there is something different: parentheses for conditions are not mandatory.
Loops are really useful in general. They do operations over and over for us. Go takes an interesting approach to these constructs by having just one single construct: for. But there are many variations of for, making Go’s instance really stand out. At times you can define the loop variable and step in the loop declaration, and you can do that outside, and if you want to execute an infinite loop, just don’t put anything after for.

I:=1
for I <= 3 {
thing
I = I + 1
}
For j:=7; j<=9; j ++ {
thing to repeat
}
for {
thing
break
}

Infinite loops would just break the program without loop control statements, which alter the normal loop execution. There are two kinds of those: break, which completely stops the execution of the loop, and continue, which skips over the rest of the rest of the below code and goes right to the next iteration.
In Go, an array is a numbered sequence of elements of a specific length. We define arrays like so:

var a [5]int
b := [5]int{1, 2, 3, 4, 5}

,where 5 is the size of the array, int is the type of the elements in the array, and in curly brackets we define the elements themselves. To also note, elements in arrays are numbered starting from 0. To access an element you put the index of it in curly brackets beside the array. To get the array length, you use the built-in len method.

Len(arr)
arr[2]

Now let’s move onto the project-making part. The project we’re going to write with Go is going to be a simple pong game, and it’s also going to be served, and to do that we’re going to use the GopherJS compiler and the GoDOM library, which was created to make the former easier-to-use and add some features to make working with it less different from working with JavaScript by itself. Notice how I said GopherJS was a compiler. Well, that means that you write some code in Go, then you go build it all, and then, during the building process the compiler converts the code you’ve written into JS code and drops a file in the project folder. Mind-blowing, I know. Now onto the project: let’s start off by creating a basic page, on which the actual game will be located. On it we should have the two racquets (or just blocks), a ball, a divider and the scoreboard. They will be created as div elements. Also, don’t forget to link the automatically-generated JS file for our script to actually work. So, the 'body' part of the HTML file should look something like this:

< div id="block">< /div>
< div id="block_r">< /div>
< div id="ball">< /div>
< div id="container_1">
< div id="score">0< /div>
< div id="score_r">0< /div>
< /div>
< div id="divider">< /div>
< script src="script.js">< /script>

As any self-respecting and totally professional web developer would do, I wrote an additional script which fixes the right block in place.

< script>
var b = document.getElementById("block_r");
setInterval(function () {
b.style.left = window.innerWidth - 35 + "px";
},100)
< /script>

And finally, add a link to a stylesheet, located in main.css in the head element like so:

< link rel="stylesheet" href="main.css">

Moving onto the script-writing phase. First of all, we should declare some variables that are going to be frequently used in func main(), like the dimensions of the screen and ball coordinates.

var iw, ih= js.Global.Get("innerWidth").Int(), js.Global.Get("innerHeight").Int()
var left, top= iw/2, ih/2
ball := Document.QuerySelector("#ball")
f := Document.QuerySelector("#block")
g := Document.QuerySelector("#block_r")
scA := Document.QuerySelector("#score")
scB := Document.QuerySelector("#score_r")
scoreA := scA.Get("innerHTML").Int()
scoreB := scB.Get("innerHTML").Int()
dir := 1
dirY := 1
var iter= false

To break the questionable parts of this down, js.Global is exactly the same as document.body in JavaScript. But then what is Document here? Well, it’s also document.body, but the former is the vanilla GopherJS syntax and the latter is the GoDOM syntax, which might be confusing, but it’s for the better, trust me. QuerySelector the GJS version of document.getElementBbyId(), the difference between those being that QS works on classes and tag names as well. The scoreA and B variables were created to keep track of the current score and are equal to the contents of the according elements, which is what innerHTML is for. The dir variables are going to be used to control the direction of the ball’s movement, and iter is a placeholder boolean which will help rid us of unnecessary actions.
Now we have to set up a function which will run periodically, but will not impede the execution of code. To do that we’re going to use goroutines, which are a lot like your regular functions, but with a small difference: these at the same time as the next function, in the background. Then we’ll put an infinite loop inside of it, and if it wasn’t for goroutines, the code after this loop would never be executed.

go func() {
for true {

First thing that we’re going to have to do is update our variables, namely the screen dimensions, the positions of blocks. We’re also going to update the ball’s position variables but only after the first cycle of the loop.

iw = js.Global.Get("innerWidth").Int()
ih = js.Global.Get("innerHeight").Int()
topA := f.Get("style").Get("top").Int()
topB := g.Get("style").Get("top").Int()
if iter {
left = ball.Get("style").Get("left").Int()
top = ball.Get("style").Get("top").Int()
}

Now we work on the logic of the game. If our ball hits the border of the screen or a racquet, it should move in a different direction and if it hit a wall, the score should change accordingly.

if left > iw-23 || left < 0 || (left <= 32 && top >= topA-5 && top <= topA+125) || (left >= iw-45 && top >= topB-10 && top <= topB+125) {
if left <= 0 {
scoreB = scoreB + 1
scB.SetInnerHTML(strconv.Itoa(scoreB))
} else if left >= iw-23 {
scoreA = scoreA + 1
scA.SetInnerHTML(strconv.Itoa(scoreA))
}
dir = dir * -1
}

And, of course, don’t forget to tell the ball to change its direction when it hit the top or the bottom border.

if (top > ih-23 || top < 0) {
dirY = dirY * -1
}

You probably noticed that earlier on, I added .Int() after using innerHTML, and now there’s this strconv string, which I never addressed. Well, innerHTML comes in the form of a string, and since I want to add to score variables, I have to convert the received value to string. The strconv library, using the Itoa method can do the exact opposite. Now we work on how the ball moves. That’s when our direction variables really come into play. To move the ball we just add the value of the dir variables which can be either 1 or -1.

ball.Get("style").Set("left", strconv.Itoa(left+dir))
ball.Get("style").Set("top", strconv.Itoa(top+dirY))

Then we set iter to true so we can update the ball’s position during the next cycle and put a small delay for smoother processing using the time library and its Sleep method.

iter = true
time.Sleep(time.Nanosecond)
}
}()

Now we make the blocks move when we press the arrow keys and W and S keys. We’re going to do that using an event listener which runs a function whenever an action is performed, in this case the action being a key press. To perform actions upon pressing these exact keys we’re going to need their key codes, which are 38, 40, 87 and 83 accordingly. We also need to consider the fact that our blocks shouldn’t move when they are too low or too high, so we have to check if their position allows for that.

Document.AddEventListener("keydown", func(e Event) {
var tp= f.Get("style").Get("top").Int()
var tp1= g.Get("style").Get("top").Int()
if e.KeyCode() == 38 && tp > -5 {
f.Get("style").Set("top", (strconv.Itoa(tp-10) + "px"))
}
if e.KeyCode() == 40 && tp < (ih-125) {
f.Get("style").Set("top", (strconv.Itoa(tp+10) + "px"))
}
if e.KeyCode() == 87 && tp1 > -5 {
g.Get("style").Set("top", (strconv.Itoa(tp1-10) + "px"))
}
if e.KeyCode() == 83 && tp1 < (ih-125) {
g.Get("style").Set("top", (strconv.Itoa(tp1+10) + "px"))
}
})
}

And that’s the end of our script. Now we just have to build it and we’ll have a ready-to-run JS file created with the power of Zeus, gopherJS and our code. Now you have created your first full-on Go project for the web!
