The Scala tutorial

Scala was developed to combine object-oriented and functional programming into one high-level language. It’s mainly used for web and desktop applications and data analysis. You may have to Scala some mountain walls that are Scala’s learning curve, but it has a very readable syntax, and it’s pretty fast. What else could a language possibly need?

To start programming with Scala, you’ll need to make sure you have all the necessary things installed, like the compiler, which is available on the official Scala website, and an IDE, which can be either Eclipse, but if you aren’t planning to use it for any other language, I’d recommend something less heavy-duty, like the official Scala IDE, available for free.

It’s a common tradition for programmers to start off their experience with a language by printing out “Hello World” in the console. Let’s jump right in!

def main (args: Array[String]): Unit = {

println("Hello, world!")

}

Let’s break this code down. Def is the basic function definition syntax, main is its name, the arguments are listed in parentheses, and the block of function code is put in curly brackets. The main function is the one that is executed first in the project. Unit is the placeholder for the function’s return value, and in this case it simply means that the function doesn’t return anything. Println is the function that does the printing out, and you don’t put semicolons at the end of inline statements, unlike most languages.

Now onto variables in Scala. A lot of more modern languages are moving on from type definition of variables, and Scala is doing so too. You can, of course, specify it anyways, but it’s by no means mandatory. Var declares a variable and val declares a constant. To define an array, which is a series of elements of the same type placed in certain memory locations, equate a variable to a new array as shown below. To access abovementioned memory locations, place the index of the element in square brackets right next to the array name.

var myVar = 10;

var myVar1 : String = "Foo"

var z = new Array[String](3)

User input is the main means of user – computer communication. The most modern way to accept user input in Scala is to use the trusty read*Type*() function like so:

var myVar = scala.io.StdIn.readInt()

var myV = scala.io.StdIn.readLine()

Moving right onto conditionals. They’re mostly self-explanatory. If a certain condition is true, an action is performed, if not, a slightly different action is performed. The implementation of if statements into Scala looks like this:

if(condition) {

do a thing

} else if (condition || condition) {

do a thing

} else if (condition && condition) {

do a thing

} else {

do a thing

}

Loops are great. They make our life easier time and time again. Wherever they’re implemented, be it an esoteric, practical or English language, they’re there to make things practical. In Scala, there are two kinds of loops – for and while. With for loops you can efficiently write a cycle that needs to be executed a specific number of times, depending on the set range, and with while loops cycle while given condition is true. There is a certain way to change the way the loop goes and that’s by using loop control statements, such as continue, which skips the rest of the loop and goes onto the next iteration, and break, which stops the execution of the cycle completely.

while(condition) {

things to do

}

for( var x <- Range ){

things to do

}

As you remember, the basic Julia program always has def main() as the main function or void. Let’s have a bit more in-depth look at functions in general. Def is the keyword that is like var for voids, then comes the name, then come the arguments, that are put in parentheses, which are values that are sent to the function from the caller. Then comes a colon and the type of data that the def is supposed to return. Then comes the = sign and then finally you put the actions that the function needs to perform in curly brackets. And that’s how functions are declared in Scala.

def addInt( a:Int, b:Int ) : Int = {

var sum:Int = 0

sum = a + b

return sum

}

Let’s make a project. A web project. We’re going to need lift, a scala framework for it. To start off type in sbt new lift/basic-app.g8 in your console. Then open up the html file in the webapp folder and the helloworld.scala file. Take a closer look a the latter. Then delete everything within the class because that’s how constructive and radical we are. To run a server and therefore get the script to work you have to run ~jetty:start in your sbt console. One of the most defining features of lift is setting css and innerHTML with one simple command. To get the HTML to change you’ll have to add a data-lift attribute to the element with its contents being ClassName.MethodName. And in the said method you put “identifier” #> “new value” where identifier can be literally anything: a tag name, an id, a class, you name it! To change an attribute put its name in square brackets after the identifier, to remove the contents of it add an exclamation point to its name and to add add a plus sign.

And that’s all the important things you could possibly need to know about lift. Congratulations, you beat Scala!
