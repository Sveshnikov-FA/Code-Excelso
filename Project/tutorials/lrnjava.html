The Java tutorial

Java is a different name for coffee, which comes from an Island of Java, located near Indonesia. It also happens to be a name of a programming language released in 1995. Its main purpose is developing mobile, web and desktop applications. Despite the language being named after coffee, it has its weaker sides, the main one being the rather bulky syntax, which doesn’t exactly fit numeric and scientific programming. But after you figure out the basic structure of the syntax, the learning curve smoothens a lot. Other major advantages include the impressive performance and the wide variety of applications you can use the language for.
To start off, you need to check if you have the compiler and IDE installed. The compiler for Java is JDK, which you can get here: *--*. The most productive IDE for Java is Eclipse.
It’s a common tradition to start learning a language by printing out “Hello, World!” in the console, so let’s jump right in.

import java.util.*;
class Main {
public static void main(String[] args) {
System.out.println("Hello world!");
} }

Let’s break this code down. Starting from the top, import is the key word for importing extra packages into the project to use. The thing that you actually want to import is defined in a fashion similar to that of websites’: if you want to specify what package to import, you keep nesting imports, so, Instead of importing the whole util package (the star at the end of java.util.* means that you import everything that is inside the package, same works for other directories), you can specify by adding, for example, Scanner, at the end, like java.util.Scanner; Class Main is the class of the main project file, the far superior one compared to other files. There can be many classes in a single file, but we’ll talk about that later. Moving on, public static void main(String[] args) is the main void of the file. There can be only one of this and it’s the part of the code that runs right after the imports, no matter the classes and voids above it(?). The basic structure of this function is simple, and all of the IDs used here can be used when declaring your very own functions. Public means it’s visible to every file of the project and all classes of the file, static means it’s going to be there for the compiler when it’s hard, or raining, or snowing, until the last string of code is executed, void simply means it’s a function, main is the name of the function, and in the brackets you see these things called arguments. There can be many of them, and there can be none (in that case the brackets should still be there). You define them like you would a variable, onto which we’ll move on later. I’ll just mention that the System.out.println() part is the one that actually is responsible for printing the code out. If you want to print out a number or a variable, there is no need to use brackets, and if you want to print out some words, then put them in single or double brackets. And if you want do mix things up a little and print out BOTH a variable and a string of words then put the different types together with a + sign between them. There’s also the super-secret codes that only mega advanced programmer people use, but I’ll share my limited knowledge with you. Basically, there are certain strings that do cool things, for example, “\n” gets you to a new line, “\b” serves as backspace (but, you know, you can still use your keyboard key), “\”” will put a quote in your string, which is handy, considering the fact that if you were to try and put it without the backslash, input would end on it, and you would more than likely get an error message, saying your coffee spilled over the mainframe or something.
Onto the next topic, variables. Java has them. As I said earlier, they’re very important, so I’ll just list most of them. Int and long are numbers sized accordingly, float and double are floating-point numbers where double can take bigger values than float, char is a single character, String is many characters and Boolean is either true or false. As said earlier variables can be parameters for voids, just like a String args[] is a parameter for main. They are defined in the same manner as shown above: *type* *name*. If there are multiple variables you would want to put in the void, then list them off with a comma as a divider. Also, when naming your variables, remember that Java is case-sensitive. That means that ab and AB are to be considered two different vars.
User input is the way to get things into the computer during the program execution. It is implemented with the abovementioned Scanner import. To start off, you need to actually import the Scanner like shown before, then you define a new scanner for the program like this, much like you would an actual scanner. And only then you can input variables accordingly.

import java.util.*;
class sample {}
class Main {
public static void main(String[] args) {
Scanner s = new Scanner(System.in);
int a = s.nextInt();
System.out.println(a);
} }

Next up are conditionals. They are defined simply as if(condition) action if true; conditions can be defined as equations (A==5) or as true statements (b) where b is a Boolean. If there is more than one action to be made, then list them off in curly brackets. And if you want the program to do something if the condition is false, there is an else parameter, which you put after the if statement, and which can also have its own condition and else.

if(a==5) {
doThings();
} else if(a==4) {
AllOfThem();
} else
doSomething();

Loops are a crucial part of programming. They make our life easier and our code – more efficient. There are two kinds of loops in java, which is unsurprising, considering the fact that not that many really work with coffee: for and while. They work at approximately the same speed, except in the case of for, everything is simply more organized than with while. The syntax is quite common.

for(int i=0;i int i=0;
while(i

Arrays in Java are rows of data of the same type. To define one you specify the type of data in it, its name and the size of it in square brackets. It’s also optional to specify its contains right away. Reminder: elements in arrays are numbered starting from 0.

int[] cats = new int[10];
int[] mice = {4, 8, 10, 12, 16};

To get the length of an array you can use the built-in length feature, and to access a particular element in it, put its index into square brackets beside the array name. There also is a way to implement dynamic arrays, the size of which you can change after definition. It is done with a thing called ArrayList, which is defined like so:

ArrayList catNames = new ArrayList();

It has its own methods, size() being the one to check its length and add() to add an extra element, thus resizing an array, the same being for remove().
Now let’s make our first Java project. It’s not going to be any ordinary project with consoles and “You have to download 72 apps for this to work” kind of thing, no-no-no, this is going to be a full-on desktop application, where we’ll have sprites of basic coffee ingredients and by mixing them together we’ll be able to make a delicious caffeine-filled drink, which does mean that there’s some theoretical parts to explain, so let’s get right into it. The two very important classes to remember here are JFrame and JPanel, and they are responsible for the window’s frame (you know, the part with the close button) and the panel (the part inside the frame). To properly tell the compiler that we’re going to be using these classes, use the extends suffix. Considering the fact that a panel cannot exist without a frame, we put the JPanel class inside the JFrame one. To set up our window we have to write some necessary code (like setting the window’s size, add the panel to it, make it visible and resizable, etc.) in void main. So that’s what we should end up with.

public class Main extends JFrame {
public static class Main1 extends JPanel {}
public static void main(String[] args) {
JFrame f = new JFrame("coffee like java");
f.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
Main1 m = new Main1();
f.add(m);
f.setSize(900,700);
f.setResizable(false);
f.setVisible(true);
} }
