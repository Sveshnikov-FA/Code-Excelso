The Python tutorial

Hissssssssssssssssssssssss… That’s the language of pythons. But thereia also is a programming language called python and it has little to do with snake tongue. It was founded by N in N, and many consider it to be the perfect language. Its easy-to-understand syntax is what gets people interested, and its efficiency is what everyone sticks around for. For such a simplistic language it has a remarkably wide range of applications: from simple numeric operations to machine learning algorithms and to operation system development.

To start off, let’s download the compiler for Python, provided on the official website here. It’s platform-independent, so that’s one less thing to worry about. And on the IDE side, the IDEal choice would be the basic, but trusty one called PyCharm, which you can get here.

It’s a customary tradition to start learning to program in a certain language by printing out “Hello, World!” in the console. So let’s just jump right into it:

print “Hello, World!”

Pretty self-explanatory, isn’t it? One thing to note here is that, as you can see, in Python you don’t put semicolons at the end of the line of code, which sets it apart from the others. And in order to receive user input, use input():

variable = input()

Let’s talk variables now. Again, going against contrary standards, Python does not, in fact, have a particularily large set of them. The type is usually automatically set during definition, so all you need to declare a variable is just name = value In python there are a variety of different collections, or numbered arrays of values of the same type. The most basic and practical one is List, a dynamic instance of arrays, which means you can modify the size of it after the initial declaration. To directly address an element, put the index, or number, of the element in square brackets beside the list name. But remember, elements in arrays in every language are numbered starting from 0! To add an extra element, thus resizing the list, use the append() method, and to do the opposite, use the remove() method, like so:

thislist = ["apple", "blueberry", "cherry"]

thislist[1] = "banana"

thislist.append("damson")

thislist.remove("banana")

Conditionals are a huge part of programming. Every language has them, but, taking into account the amount of liberties Python’s syntax takes, it’s kind of weird they left them in. They are defined like this:

if test1:

state1

elif ((test2 and test3) or test4 or test5):

state2

else:

state3

As you can see, there are some unique things about these rather simple lines of code, which help capitalize on Pythons originality. First of all, the conditions themselves aren’t stated in brackets, but it’s advised to do so if there is more than one condition for the sake of readability. In python, the roles of logical and, or are played by, shocker, words and, or. But, going back to general syntax analysis, the colon is here to take the place of the curly brackets in other languages. Moreover, instead of else if, Python implements a more compact line elif which serves the same purpose: if test1 is not true then check if test2 is true.

Let’s address the indentation in python, which you may have noticed in the if/else example. Unlike the other languages, in which it’s usually just a readability optimization practice, here it’s a very important part of the syntax. Whenever there are colons placed, to tell the compiler that this is the code that you want to be executed upon these conditions, you need to indent the needed code by pressing tab so the initial if statement is one tab to the left of the code. This rule is mandatory for every code block, whether it’s a conditional, a loop or a function.

Another important part of programming are loops. They make our life easier and our code more efficient. Python decided to keep the two common types of loops: for and while, but he did change up their definition in his usual style. for x in range(25):

print x

while a <= b:

print a

So, the first thing to address is the range operator in for. What is that, you might ask? Well, it is an operator that gives in the range in which the variable is going to be used. You can use other variables as limits, and you can also set the starting point of the range like this: range(1,5) ( if you don’t the compiler puts 0 in by default ).

string = "Hello World"

for x in string:

print x

A function, also known as a void, is a block of code that’s only run when explicitly called. To declare one use the command word def, then the function name, then list the arguments, the values you choose to send as extra data to the void, in parentheses, then put the colon to mark the beginning of a function. The said block of code is to be placed with required indentation, as usual.

def my_function(value):

print(“Hello, world”)

But what’s calling a function? Exactly what it means in the English language: you just call out its name, and if there are arguments to be sent, then it’s like calling someone by their full name.

my_function(5)

There is also a noteworthy thing such as default argument values, which means that if you declare the argument variable within the function definition, you’re allowed to not send any arguments when calling said void.
To stop every function ever, you can use quit().
Python’s packages cannot go unaddressed. The more well-known ones are numpy, which is the most extensive scientific computing and data analysis package out there, winsound, which can, well, make sound on windows machines when needed, and curses, which is a package for working with the terminal interface. Let’s go a bit more in-depth on the last two.
Winsound provides access to the basic sound-playing machinery provided by Windows platforms. To make a simple beep with the PCs speaker, you can use the winsound.Beep(freq, tl) function, but, of course, you’ll have to specify the frequency of the sound and the duration of the beep yourself. The PlaySound(sound,flags) function lets you play any sound you want: if you want to play your own audiofile, the flag must be winsound.SND_FILENAME and if you want to play a system sound (we’ll get back to those in a bit) use winsound.SND_ALIAS. And the MessageBeep(type) function lets you play a system sound directly. But what is a system sound in this case, you might ask? Well, a good example of one would be the Windows 10 notification noise that you might hear when Windows Defender is done trying its best. The sound in question will be played accordingly to the used constant, the full list of which goes like so: winsound.MB_OK, which is the default system sound, winsound.MB_ICONASTERISK, which is like MB_OK but not really, MB_ICONEXCLAMATION which plays a SystemExclamation sound, MB_ICONHAND, which plays the SystemHand sound, and MB_ICONQUESTION, which plays the SystemQuestion sound. Curses provide access to the terminal User Interface. With curses you can do fun things like make animations with ASCII characters, change the color of text in the terminal, and make games! Let’s initialize the library by using curses.initscr() and also reserve the space in the terminal. If we declare a variable that’s equal to intiscr, we can get the screen’s properties, like its height and width ( which is done inline using varname.getmaxyx() ). Going on with setting things up, let’s configure the cursor’s state, like if it;’s going to be visible or not, by using curses.curs_set(visibility_level), where vl can be set to 0, 1 or 2 and 2 is “very visible”, which means that the cursor will be shown as a large white block. To prepare curses for working with color, you need 2 lines of code: curses.start_color() and curses.use_default_colors(), the latter meaning we’ll address colors using the curses-provided constants. Moving on, let’s finish setting up the actual window by using newwin(lines, cols, y, x) where lines and cols are the height and width of the widow respectedly and y and x are the coordinates of the left-upper corner of it. Finally let’s set up user input in curses by using window.keypad(1). There’s also ways to refresh the window every once in a while using window.timeout(time) or window.refresh() (this one will not repeat by itself).
Now that we’re finally done setting everything up, let’s start working with the window. You can add single characters at desired locations(within reason and bounds of the window) using addch(y, x, character) or even whole strings of them using addstr(y, x, string). But what if you don’t want to look up the needed character to paste it in just for the compilator to tell you that nothing works? Well, curses got you covered with their extensive constant collection of ASCII characters, which are initialized like this: curses.ACS_KEY. This should work, but it doesn’t look fancy enough. Well, we can change the color of the text, by adding a comma after the value and putting in curses.COLOR_colorname. Color too vibrant for you? You can mix colors here with curses.init_pair(id,color1,color2) and then reference the pair by curses.color_pair(id). Also worth mentioning that curses has its own input methods, where window.getch() receives a single input key, which, by the way, may not only be a letter or a number – it can be any key that you have on your keyboard, and window.getstr() receives a whole string. Going back to the whole keyboard input thing, curses still got you covered with their constant system, which, this time, contains names, by which you can reference non-character keys, like the Left-Arrow key, which is curses.KEY_LEFT in the constant library. If you, for whatever reason, want to stop curses, use curses.endwin().

w = curses.newwin(screenHeight, screenWidth, 0, 0)
w.keypad(1)
