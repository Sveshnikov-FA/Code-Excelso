The Kotlin tutorial

Kotlin is a fairly new programming language developed by JetBrains to be safer and more minimal than Java and simpler than Scala. Its mainly used for desktop and mobile development, and while the performance may be a bit of a turn-off, Kotlin serves its purpose well: it has backwards and cross-platform compatibility and its syntax is very efficient.
To start off let’s make sure you have everything necessary installed. To code in Kotlin you’ll need a compiler, available on the official website, and an IDE, the role of which can be fulfilled by the well-known Eclipse IDE.
It’s a customary tradition among programmers to start learning a new language by printing out “Hello, world!”, so let’s do exactly that.

fun main(args: Array) {
println("Hello World!")
}

Let’s break this code down. Fun is the way to define functions in Kotlin, main is the name of the function and is the first function to be executed upon the launching of the code. In parentheses we put arguments, the variables we send over to the function from the main one, and in this case, considering the fact that this IS the main function, the arguments are obligatory and should be the same in every Kotlin file. The actions that are supposed to be executed when function is called are put in curly brackets, but as we see below, there is no semicolon to be found at the end of the println line, the one that does the printing out of the hellos and the worlds.
Now, it’s important to know how variables work in a language, and the more modern ones choose to let go of the type declaration, and Kotlin is no exception. Well, that’s not entirely true, there is a way to declare variable types, but you can tell everyone’s trying to move on from that technique, meaning that:

val firstName: String = "Chike"
and
val firstName = "Chike"

do exactly the same thing. But anyways, val varName is the way to declare constants, and if you want to declare a variable, use var instead. But what will we do if we want to declare an incredibly large number as a variable? Well, fear not, you won’t exactly have to come back to the direct type declaration, and instead, we are going to use a shorter form:

val myLong = 19L

Just a reminder: this works the same way the full type declaration does, like this:

val myLongAgain: Long = 40

Another interesting feature of the variables to note is that you can put underscores in the value upon declaration: the compiler will completely ignore them, and the code becomes more readable.

val oneMillion = 1_000_000

Arrays are a numbered sequence of elements of a specific length. To declare one we must use the arrayOf function and list the elements.

val myArray = arrayOf(1, 2, 3, 4, 5)

Looks simple,right? Well here’s something very confusing for you:

val myArray = arrayOf(1, 2, 3, 4, 5, “what”)

Believe it or not, this will compile just fine. It’s just something called a mixed-type array. But if you want to be more clear with the types of elements in the array, specify the type in angle brackets, or instead use typeArrayOf().

val myArray = arrayOf(1, 2, 3, 4, 5)

Now let’s try to get user input. There are many ways to do that in Kotlin, and I’ll only address two of the most practical ones. The first one is done using the built-in and oh-so-familiar readline() function which works rather questionably:

val (a, b) = readLine()!!.split(' ')
println(a.toInt() + b.toInt())

This means that redline gets only string input, and you have to manually split the string in two to get this to work with two variables. It works just fine without that if you only want one variable to be input, but this syntax will definitely take some getting used to. So here’s a different way to get input:

import java.util.Scanner
fun main(vararg args: String) {
val input = Scanner(System.`in`)
val a = input.nextInt()
val b = input.nextInt()
println(a + b)
}

This one is certainly more organized than the first one but it does have an import required, which could be confusing if your project has many of those kinds of statements.
Conditionals are implemented in a rather simple way in Kotlin. They look and work like this:

if (condition) { do a thing } else { Do a slightly different thing }

Loops are great. They help free us from the extra and repetitive work by repeating the same code a number of times until a loop control statement is false. There are 2 types of loops: for and while. With for you can define the range in which the loop variable is contained. There are several ways to declare for loops: a simple range:

for (i in 1..3) { println(i) }

A range from up to down with a modified step:

for (i in 6 downTo 0 step 2) { println(i) }

A range in an array, which goes from 0 to (array length)-1

for (item in collection) { print(item) }

And the while loop only repeats while the loop control statement is true, no initial value, no step defined within the loop declaration.

while (condition) { do a thing }

Sometimes loops iterate for way too long, and the way to handle that are loop control statements, which alter the normal loop iteration. There are also two loop control statements: break, which completely stops the loop execution, and continue, which just skips the rest of the loop after it and moves onto the next iteration. Now onto the project-making part. We’re going to make a full-on desktop application about making coffee, like real big-boy programmers. If you followed my Java course, this should sound and look familiar (just like everything about Kotlin, go figure). To set up our window we’ll need a scene and stage, which are responsible for the frame of the window and its contents accordingly. The windows setup is going to be located in a separate class which extends the package Application. Then we launch the window using an according command and setup some necessary things (like window size, its root, resizability and visibility) within a new function called start which takes stage as a parameter. You should end up with something like this.

class Main: Application() {
override fun start(stage: Stage) {
var scene: Scene = Scene(root, 955.0,700.0, Color.WHITE)
scene.root = root
stage.scene = scene
stage.title = "coffee game v2"
stage.isResizable = false
stage.show()
} }
fun main(args: Array) {
Application.launch(Main::class.java, *args)
}

You could say that the start function is our stage of action. Instead of adding a separate function for a mouse listener, Kotlin provides a steady method which we’ll use to update our ingredients array like so:

var lungoInit = ImageView(lungo)
lungoInit.x = 550.0
lungoInit.y = 219.0
lungoInit.setOnMouseClicked {
coffee.add("lungo")
}

After initializing the rest of the objects, we’ll set up the functions for the buttons. When we click on the back button, we should clear our array, remove anything that shouldn’t be there (by just grouping every single object onscreen and deleting the last one) and wipe the sevice variables.

var backInit = ImageView(back)
var enterInit = ImageView(enter)
var root = Group(tableInit,lungoInit,kettleInit,whiskInit,milkInit,ristrInit,cpotInit,enterInit,backInit,wcInit,teaInit,iceInit,lemonInit,lab)
backInit.x = 10.0
backInit.y = 10.0
backInit.setOnMouseClicked {
coffee.clear()
if(root.children.last()!=lab) root.children.remove(root.children.last())
wh = 0
lab.text = ""
}
